Creating and using packages

In Python, packages are a way to organize related modules into a hierarchical directory structure. A package is essentially a directory that contains multiple Python module files and an additional __init__.py file that marks the directory as a package.

Creating a Package

To create a package, you need to create a directory with a meaningful name. Inside this directory, you can include multiple Python module files and an __init__.py file. 


# Create a package named my_package with the following structure

my_package/
├── __init__.py
├── module1.py
└── module2.py

In this example, the my_package directory is created and contains an __init__.py file along with two module files, module1.py and module2.py.

Using Modules from a Package

Once you have created your package, you can import and use modules from it using the dot notation. 


import my_package.module1
from my_package import module2

my_package.module1.function1()
result = module2.function2()

In this code, the module1 module is imported from the my_package package using the dot notation. The module2 module is imported using the from keyword. The functions function1() from module1 and function2() from module2 are called accordingly.

Importing Modules with Package Alias

You can import modules from a package with an alias (alternate name) using the as keyword.


import my_package.module1 as mod1
from my_package import module2 as mod2

mod1.function1()
result = mod2.function2()

In this code, the module1 module is imported with the alias mod1, and the module2 module is imported with the alias mod2. The functions from the modules are accessed using the respective aliases.

Using Sub-packages

Packages can also contain sub-packages, which are simply nested directories with their own __init__.py files. This allows for further organization and structuring of code. 


#  Expand the my_package package structure to include a sub-package named 
my_package/
├── __init__.py
├── module1.py
├── module2.py
└── sub_package/
    ├── __init__.py
    └── module3.py

In this example, the sub_package directory is created inside the my_package package and contains its own __init__.py file along with a module file, module3.py. The sub-package can be imported and used in a similar manner as the main package.
